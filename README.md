# it-support-playbooks

Coleção de guias de troubleshooting pra problemas recorrentes de suporte técnico corporativo (N1/N2). Cada guia é um repositório próprio, com o mesmo padrão: sintoma, causa provável, como confirmar, solução e como evitar de novo.

Desenvolvido por **Davi Senise, Técnico de Suporte TI**.

## Por que esses guias existem

Chamado de suporte tende a se repetir. O mesmo tipo de problema chega de formas diferentes, mas a causa raiz costuma ser parecida. Documentar esses padrões reduz o tempo de diagnóstico e serve de referência tanto pra mim quanto pra qualquer pessoa que precise resolver o mesmo tipo de chamado.

Cada guia segue a mesma estrutura de raciocínio:

1. Qual é o sintoma relatado
2. Qual é a causa mais provável
3. Como confirmar essa causa antes de agir
4. Qual é a solução
5. Como evitar que o mesmo problema volte a acontecer

## Como usar

Cada repo é independente e pode ser consultado direto, sem depender dos outros. Se você trabalha com suporte técnico, sinta-se livre pra adaptar qualquer um deles pro seu ambiente.
